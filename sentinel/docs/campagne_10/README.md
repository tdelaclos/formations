# Campagne 10 — RPM et cycle de vie

Cette campagne transforme Sentinel en composant natif du socle AlmaLinux. Elle couvre toute la chaîne : construire, déclarer les dépendances, préserver la configuration, signer, publier et exploiter le paquet.

## Parcours

1. [Construire un paquet RPM](10.1-construire-paquet-rpm.md) ;
2. [gérer les dépendances RPM](10.2-gerer-dependances-rpm.md) ;
3. [gérer les fichiers de configuration RPM](10.3-gerer-fichiers-configuration-rpm.md) ;
4. [signer les paquets RPM](10.4-signer-paquets-rpm.md) ;
5. [exploiter un dépôt RPM privé](10.5-exploiter-depot-rpm-prive.md) ;
6. [packager Sentinel](10.6-packager-sentinel.md).

## Résultat attendu

À la fin de la campagne, l'apprenant dispose d'un RPM Sentinel signé, qualifié sur une VM AlmaLinux propre et distribué par un dépôt DNF privé avec vérification des paquets et des métadonnées.

Les chapitres suivent le formalisme défini dans le [`GUIDE-REDACTION.md`](../GUIDE-REDACTION.md).
